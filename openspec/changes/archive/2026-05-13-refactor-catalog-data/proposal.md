## Why

為了讓產品目錄的結構更符合真實的業務分類，並且讓系統能夠由單一資料源自動驅動全站的產品顯示，我們需要將靜態的 Mock 資料替換為由 `AZX產品型錄.xlsx` 轉換而來的 JSON 資料。此舉能有效降低未來資料維護的成本，並解決同名產品多重規格顯示的問題。

## What Changes

- **固化大類分類**：產品目錄固定為 16 個大項。
- **重構子類別**：將現有寫死的子類別（如辦公桌、人體工學椅等）註解隱藏。僅針對「機械及零配件」大類新增其專屬的 8 個子類別。
- **動態分類導航**：若某個產品大類下沒有定義子類別，當使用者點擊該大類時，將直接在頁面上顯示該大類下的所有產品清單。
- **資料轉換與結構統一**：
  - 將 `AZX產品型錄.xlsx` 內的有效資料轉換為固定的 JSON Object 格式（忽略特定的範例列），且轉換後 JSON 物件的所有屬性名稱 (Key) 皆須為英文。
  - 在轉換過程中，忽略 XLSX 內的「數量」與「序」欄位，不將其納入最終的 JSON Object 結構中。
  - 對於空欄位保留 Key 並賦予 Empty String (`""`)。
  - 當產品缺少圖片連結時，網頁預覽將顯示「圖片待更新」。
- **同名產品規格合併**：在轉換或讀取 JSON 過程時，將名稱相同的資料列視為同一個產品的不同規格（例如：搖頭棘輪梅開板手有 3 筆資料 -> 合併為 1 個產品物件，內含 3 個規格項目）。網頁端的明細頁面將根據此結構動態切換規格。

## Capabilities

### New Capabilities
- `product-catalog-refactor`: 包含分類結構重構、XLSX 到 JSON 的資料源轉換、同名產品合併，以及對應的分類頁面渲染邏輯修改。

## Impact

- `src/pages/Category/*` & `src/pages/Subcategory/*`: 分類點擊與產品列表的渲染邏輯。
- `src/pages/Product/Product.tsx`: 產品規格資料的載入與切換。
- 資料來源層：新增 JSON 轉換腳本與統一的資料讀取介面。
