## Context (背景脈絡)

目前的網頁平台缺乏一致的響應式網頁設計 (RWD) 行為，特別是在行動裝置上。當使用者使用如 iPhone Pro 與 Pro Max 等現代智慧型手機造訪網站時，可能會遇到文字被切斷、出現水平捲軸以及元素重疊等問題。實作穩健的 RWD 可以確保當視窗變窄時，UI 元件能優雅地流動與堆疊。

## Goals / Non-Goals (目標 / 非目標)

**Goals (目標):**
- 為 `/src` 內的所有頁面（首頁、次分類、產品明細、公告、關於我們、隱私權政策、服務條款等）提供響應式版面。
- 確保網站版面在縮放至 390px 視窗時完美運作且不產生水平捲軸（輕鬆涵蓋 iPhone 13/14/15 Pro Max 及類似的現代智慧型手機）。
- 針對小螢幕上的觸控操作，調整較複雜的元件，例如 Header 導覽列、巨型選單 (Mega Menu) 與浮動購物車。

**Non-Goals (非目標):**
- 重新設計整個網站的視覺美學或配色方案。
- 引入漸進式網頁應用程式 (PWA) 的功能或原生 App 行為。

## Decisions (技術決策)

- **媒體查詢策略 (Media Query Strategy)**：我們將在現有的 CSS 檔案中附加行動版的媒體查詢 (`@media (max-width: 768px)` 與 `@media (max-width: 480px)`)。這能避免大幅重寫整個網格系統，同時確保桌面版的版面維持原樣不受影響。
- **Header 導覽列 (Header Navigation)**：將水平導覽選單轉換為適合行動裝置觸控的佈局。巨型選單將需要垂直堆疊，或採用手風琴 (Accordion) 樣式，以適應 430px 的寬度。
- **產品網格 (Product Grids)**：調整 `grid-template-columns`，使其在小於 768px 的螢幕上顯示為單欄 (`1fr`)。
- **浮動購物車 (Floating Cart)**：確保購物車面板的寬度在行動裝置上佔用將近 100% 的視窗，且為符合無障礙設計 (Accessibility)，數量增減的觸控目標應至少為 44x44px。

## Risks / Trade-offs (風險與權衡)

- **風險：行動版 Header 互動過於複雜** → 緩解措施：在行動版簡化巨型選單的顯示方式，將項目垂直堆疊以防止觸控目標重疊。
- **風險：圖片超出容器範圍** → 緩解措施：在全域或針對特定卡片圖片套用 `max-width: 100%; height: auto;` 以確保流暢縮放。
